  XPath Queries (Exactly 10 Required)

The following XPath expressions are designed to extract and filter
data from students.xml. Each query includes an explanation and its
expected output.

--------------------------------------------------------------------
QUERY 1: Count total number of enrolled students
--------------------------------------------------------------------

Expression:
  count(//student)

Explanation:
  Uses the count() function to return the total number of
  <student> elements in the document. Useful for enrollment
  statistics and summary reports.


--------------------------------------------------------------------
QUERY 2: Calculate the total units enrolled across all students
--------------------------------------------------------------------

Expression:
  sum(//subject/units)

Explanation:
  Uses the sum() function to aggregate the <units> values across
  all <subject> elements in the entire document. Useful for
  institutional workload and resource planning.


--------------------------------------------------------------------
QUERY 3: Retrieve all students with GPA > 2.0
(Required by Case Study)
--------------------------------------------------------------------

Expression:
  //student[gpa > 2.0]

Explanation:
  This query selects all <student> elements where the <gpa> child
  element has a value greater than 2.0. In the PLP grading system,
  a GPA above 2.0 indicates a lower academic standing (higher number
  = lower performance), so this identifies students who may need
  academic intervention.


--------------------------------------------------------------------
QUERY 4: Get the student ID and GPA of the Dean's Listers
         (GPA <= 1.75)
--------------------------------------------------------------------

Expression:
  //student[gpa <= 1.75]/@studentId | //student[gpa <= 1.75]/gpa

Explanation:
  Uses the union operator to select both the studentId attribute
  and gpa element of students whose GPA is 1.75 or below,
  qualifying them for Dean's List honors.


--------------------------------------------------------------------
QUERY 5: List all students enrolled in the BSIT program
--------------------------------------------------------------------

Expression:
  //student[program = 'BSIT']

Explanation:
  Selects all <student> elements whose <program> child element
  matches exactly "BSIT". This is useful for generating
  program-specific reports or enrollment counts.


--------------------------------------------------------------------
QUERY 6: Retrieve students from BSCS program in year level 4
--------------------------------------------------------------------

Expression:
  //student[program = 'BSCS' and yearLevel = 4]

Explanation:
  Combines two predicates using the logical "and" operator to
  filter students by both program and year level simultaneously.
  Demonstrates compound filtering.


--------------------------------------------------------------------
QUERY 7: Find all subjects with a grade of 1.00 (perfect mark)
--------------------------------------------------------------------

Expression:
  //subject[grade = 1.00]/subjectName

Explanation:
  Selects the <subjectName> of every <subject> where the <grade>
  equals 1.00, representing a perfect/highest mark. This can be
  used to identify high-performing subject areas.


--------------------------------------------------------------------
QUERY 8: Retrieve faculty details for all subjects taught by Sir
         Lito Pacquiao
--------------------------------------------------------------------

Expression:
  //subject[facultyMember/professorName = 'Sir Lito Pacquiao']

Explanation:
  Navigates into the nested <facultyMember> block to filter subjects
  by a specific professor's name. Demonstrates path traversal into
  nested elements added in the updated XML structure.


--------------------------------------------------------------------
QUERY 9: Retrieve all subject names along with their contact hours
--------------------------------------------------------------------

Expression:
  //subject/subjectName | //subject/hours

Explanation:
  Uses the union operator to retrieve both the <subjectName> and
  <hours> values from all subjects. The <hours> element was added
  in the updated XML structure to reflect contact hours.


--------------------------------------------------------------------
QUERY 10: List subjects taught by faculty from College of Computer
          Studies
--------------------------------------------------------------------

Expression:
  //subject[facultyMember/department = 'College of Computer Studies']/subjectName

Explanation:
  Filters subjects based on the faculty member's department, which
  is stored in the nested <facultyMember/department> element. Useful
  for department-level reporting.
