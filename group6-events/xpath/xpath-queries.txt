=== BASIC QUERIES ===

1. List all events (50 total)
   XPath: //event
   Purpose: Retrieve all event records from the system
   Example Result: 50 event elements with eventId attributes (EVT-001 through EVT-050)

2. List all participants (125 total)
   XPath: //participant
   Purpose: Retrieve all participant records
   Example Result: 125 participant elements with participantId attributes (PAR-001 through PAR-125)

3. Count total participants
   XPath: count(//participant)
   Purpose: Get total number of participants
   Expected Result: 125

4. Count total events
   XPath: count(//event)
   Purpose: Get total number of events
   Expected Result: 50

5. Count total registrations
   XPath: count(//registration)
   Purpose: Get total number of registrations
   Expected Result: 200

=== STATISTICAL QUERIES ===

31. Count registered vs pending registrations ratio
    XPath: Registered: count(//registration[registrationStatus='Registered']), Pending: count(//registration[registrationStatus='Pending'])
    Purpose: Get registration status breakdown
    Expected Result: ~160 Registered, ~40 Pending

32. Average registrations per event
    XPath: count(//registration) div count(//event)
    Purpose: Calculate average participants per event
    Expected Result: ~4 registrations per event

33. Most popular event (highest registrations)
    XPath: //event[count(//registration[eventId=current()/@eventId]) = max(//event/count(//registration[eventId=current()/@eventId]))]
    Purpose: Find most attended event
    Expected Result: Event with 5 registrations
